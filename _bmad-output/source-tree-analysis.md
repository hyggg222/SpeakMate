## Core Application Structure (`speakmate/`)
Built with the T3 Stack (Next.js + Drizzle + Tailwind + TypeScript).

```text
speakmate/
├── public/                     # Static assets (favicons, etc.)
├── src/
│   ├── app/                    # Next.js App Router (Pages and API routes)
│   │   ├── api/                # Backend API endpoints
│   │   │   └── safe-mode/chat/ # Main AI Chat streaming logic
│   │   ├── practice/           # UI for practice modes
│   │   └── layout.tsx          # Root layout with fonts and UI shell
│   ├── components/             # UI Components
│   │   ├── shared/             # Global components (Navbar, Footer)
│   │   └── ui/                 # shadcn/ui base components
│   ├── features/               # Modular Feature logic
│   │   └── safe-mode/          # Logic, hooks, and components for Safe Mode
│   ├── lib/                    # Shared utilities and configurations
│   │   ├── errors.ts           # Custom AppError handling
│   │   ├── supabase.ts         # Supabase Admin client configuration
│   │   └── utils.ts            # Tailwind merging and common helpers
│   ├── server/                 # Server-side logic
│   │   └── db/                 # Database schema and Drizzle connection
│   └── styles/                 # Global CSS and Tailwind configuration
├── .env                        # Environment variables
├── drizzle.config.ts           # Drizzle ORM configuration
├── next.config.js              # Next.js framework configuration
├── package.json                # Project dependencies and scripts
├── tsconfig.json               # TypeScript configuration
└── vitest.config.ts            # Testing framework configuration
```

### Critical Folders Summary
- **`src/app/api/safe-mode/chat`**: The "brain" of the AI interaction, handling multimodal streaming.
- **`src/features/safe-mode`**: Contains the core business logic for the first MVP mode.
- **`_bmad-output/planning-artifacts`**: The source of truth for all product requirements.
