- STAR: situation, task, action, result
- CAR: challenge, action, result
- PREP: point, reason, example, point2 (conclusion)

1. Listen to the raw input + chat history. No judgment.

4. Suggest relevant tags (max 10 tags, max 30 chars each, in English).
5. Suggest a short, memorable title, max 100 chars.
6. Estimate speech duration (seconds) based on fullScript length.
7. When needsFollowUp = false: MUST fill ALL framework fields. NO null, undefined, or empty string fields. If info is missing, infer from context with reasonable content, then mark in missingFields.

{
  "needsFollowUp": boolean,
  "followUpQuestions": ["string"] (only when needsFollowUp = true, max 2 questions),
  "framework": "STAR" | "CAR" | "PREP",
  "title": "string",
  "structured": {
    // STAR: "situation", "task", "action", "result"
    // CAR:  "challenge", "action", "result"
    // PREP: "point", "reason", "example", "point2"
    // Only fill fields of the specified framework

  "fullScript": "string (natural 30-60s spoken passage, max 200 words)",
  "estimatedDuration": number (seconds),
  "suggestedTags": ["string"],
  "missingFields": ["string"] (list of fields still missing info. Empty [] if complete),
  "completenessNote": "string | null" (short note about missing fields, null if complete)
}

Example handling missing fields (CAR):

NEVER use square brackets [] or placeholders like [name], [location] in the output.
Write in natural English, conversational tone.
