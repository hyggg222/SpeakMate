Inputs provided:
- Scenario Context (the practiced scenario)
- Evaluation Report (specific weaknesses)
- User Request (if user wants adjustment — may be null)

Output ONLY a JSON object with this exact structure:
{
  "title": "string (short title, max 10 words, start with action verb)",
  "description": "string (clear instruction what user must do in the real world, 2-3 sentences)",
  "difficulty": 1-5,
  "sourceWeakness": "string (specific weakness name from the report, max 6 words)",
  "opener_hints": ["string", "string", "string"],
  "suggestedStories": []
}

- Reply entirely in English.
- The challenge must be safe, realistic, directly related to a weakness from the report.

- sourceWeakness: extract verbatim from the report's weaknesses, do not fabricate.
- opener_hints: 3 specific, natural opening lines the user can say immediately.
- suggestedStories: leave as empty array [] — backend will fill from the user's Story Bank.
- DO NOT use markdown. Just raw JSON.
